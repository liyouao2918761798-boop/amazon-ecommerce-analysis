# 基于 Amazon 电商真实数据集的商业化数据分析

## 项目简介
本项目围绕 Amazon 电商场景展开，综合使用 **Amazon Sales Dataset** 与 **Amazon Product Reviews（Electronics）** 数据，对商品、评论、用户活跃度与推荐原型进行分阶段分析。项目目标不是只做基础可视化，而是从数据清洗、探索性分析、情感与主题挖掘，到推荐原型与最终商业总结，形成一个较完整的商业数据分析流程。

本仓库对应课程项目的**最终提交版本**，内容已经按照阶段整理完成，包含周报/阶段报告、处理后表格、交互式仪表板、推荐原型代码以及最终商业报告。

---

## 项目目标
本项目主要完成以下四类任务：

1. **阶段一：数据清洗与结构化处理**  
   对原始 Amazon 商品与评论数据进行探索、清洗、字段规范化和结构化整合，生成后续分析所需的主分析表与辅助表。

2. **阶段二：探索性分析与可视化**  
   从销售表现、用户活跃度、商品评分、评论长度和评论情绪等角度开展探索性分析，并制作交互式仪表板展示结果。

3. **阶段三：建模与深度洞察**  
   在评论数据上进行情感分析、主题提取，并构建简单推荐原型；同时分析推荐效果偏低的原因，并对模型表现进行对比说明。

4. **阶段四：综合商业总结**  
   汇总前面各阶段结果，形成最终商业分析报告与项目汇报材料。

---

## 数据来源说明
本项目使用的数据主要来自以下两个来源：

- **Amazon Sales Dataset**  
  用于商品基础属性、价格、折扣、评分数、评论数等字段分析。

- **Amazon Product Reviews（Electronics）**  
  用于用户评论、评分行为、评论文本分析以及推荐原型构建。

### 数据使用说明
由于两个数据集的产品标识体系并不完全一致，因此：

- 阶段一中以商品主表为核心构建主分析宽表；
- Electronics 评论数据主要用于评论分析、情感分析、主题建模与推荐原型；
- 对少量可直接匹配的产品，进行了有限整合；
- 项目中的部分指标属于**代理指标（proxy）**，用于探索性分析，不等同于真实平台财务指标。

## 方法说明

### 1. 用户活跃度分层
在缺少完整真实消费金额链路的前提下，本项目以**评论行为**作为用户参与度代理变量，按照评论次数进行经验分层：

- `One-time`：1 次
- `Light`：2–4 次
- `Active`：5–9 次
- `Power`：10 次及以上

该分层主要用于阶段二探索性分析，反映用户参与强度，而不直接代表真实消费金额价值。

### 2. 销售潜力代理指标
由于数据无法支持严格的真实 GMV 核算，项目在探索性分析中使用“价格 × 评论/评分数量”构造相对销售潜力代理指标。该指标适合用于类目内部或样本内部的相对比较，但不等同于真实财务口径 GMV。

### 3. 情感分析说明
项目使用通用文本情感工具对评论文本进行初步情绪分析，用于识别整体情绪分布趋势。需要注意：
- 电商评论中存在大量场景化表达；
- 专业术语与反讽表达会影响判断准确度；
- 该结果应理解为**探索性情绪信号**，不应视作高精度情感标签。

### 4. 推荐原型说明
阶段三的推荐原型主要用于课程项目中的方法展示与效果对比验证。项目进一步分析了推荐结果偏低的原因，包括：
- 用户-商品交互矩阵稀疏；
- 样本量与交互深度有限；
- 简单模型表达能力有限；
- 仅有单一品类交互数据，限制了跨品类对比分析。

因此，推荐结果更适合作为**实验性原型**，而不是上线级推荐系统。

---

## 主要结论概述
1. 当前数据更适合做**结构化商业探索**，而不是做严格的全链路销售归因。  
2. 不同类别在评论情感分布和主题上存在差异，说明跨品类不能简单混合解释。  
3. 用户活跃度存在明显分层，高频用户对评论与反馈结构影响较大。  
4. 推荐模型整体效果偏弱，主要受数据稀疏性和交互深度不足影响，而不只是模型本身问题。  
5. 课程项目层面已经能够展示从数据准备、分析、建模到商业总结的完整流程。

## 项目局限性
1. 两个原始数据集的产品 ID 体系不完全统一，限制了全量整合。  
2. 缺少完整真实销量、订单金额和用户购买链路，因此部分商业指标只能使用代理口径。  
3. 推荐原型仅基于有限交互数据，结果不能代表工业级推荐系统水平。  
4. 情感分析采用通用方法，对电商特定表达的理解仍有偏差风险。
